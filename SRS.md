# Software Requirements Specification (SRS) ฉบับย่อ
**โปรเจกต์:** ระบบรับจองโต๊ะร้านกาแฟ (Coffee Shop Table Booking System)

---

## 1. ภาพรวมระบบ (System Overview)
ระบบแอปพลิเคชันสำหรับจัดการการจองโต๊ะของร้านกาแฟ เพื่ออำนวยความสะดวกให้ลูกค้าสามารถเลือกวัน เวลา และโซนที่นั่งได้ด้วยตนเอง พร้อมทั้งมีระบบจัดการหลังบ้านเพื่อให้พนักงานสามารถตรวจสอบคิวและสถานะของโต๊ะต่างๆ ภายในร้านได้อย่างมีประสิทธิภาพ

---

## 2. กลุ่มผู้ใช้งาน (User Roles)
ระบบนี้รองรับผู้ใช้งาน 2 กลุ่มหลัก ได้แก่:
1. **ลูกค้า (Customer):** ผู้ใช้งานทั่วไปที่ต้องการจองโต๊ะล่วงหน้าผ่านระบบ
2. **พนักงาน/แอดมิน (Staff/Admin):** ผู้ดูแลระบบของร้านที่คอยตรวจสอบ จัดการคิว และอัปเดตสถานะการจอง

---

## 3. ฟีเจอร์ "ต้องมี" (Must-Have Features)
ฟีเจอร์พื้นฐานที่ระบบจำเป็นต้องมีเพื่อให้สามารถเปิดใช้งานจริงได้ (MVP):

**สำหรับลูกค้า (Customer):**
- [x] เลือกวันที่ และเวลา ที่ต้องการจอง
- [x] ระบุจำนวนคนที่เข้าใช้งาน
- [x] เลือกโซนที่นั่งที่ต้องการ (เช่น Indoor / Outdoor)

**สำหรับพนักงาน/แอดมิน (Staff/Admin):**
- [x] หน้าดูรายการการจองทั้งหมด (Booking List)
- [x] ฟังก์ชันในการ อนุมัติ (Approve) หรือ ยกเลิก (Reject) การจองของลูกค้า
- [x] หน้าแสดงสถานะโต๊ะ ว่าโต๊ะไหนว่าง/ไม่ว่าง ในช่วงเวลานั้นๆ

---

## 4. ฟีเจอร์ "น่ามี" (Nice-to-Have Features)
ฟีเจอร์เสริมที่สามารถพัฒนาเพิ่มเติมในเฟสต่อไป เพื่อยกระดับประสบการณ์ผู้ใช้งาน:

**สำหรับลูกค้า (Customer):**
- [ ] ระบบให้ลูกค้าสามารถสั่งเครื่องดื่มหรืออาหารล่วงหน้าได้ตั้งแต่ตอนกดจองโต๊ะ

**สำหรับพนักงาน/แอดมิน (Staff/Admin):**
- [ ] ระบบแจ้งเตือน (Notification) ไปยังพนักงานผ่านช่องทาง LINE ทันทีเมื่อมีรายการจองเข้ามาใหม่

---

## 5. ข้อมูลที่ต้องเก็บ (Data Storage)
ข้อมูลสำคัญในฐานข้อมูล (Database) ที่ระบบจำเป็นต้องบันทึก แบ่งเป็น 2 ส่วนหลักๆ ดังนี้:

### 5.1 ข้อมูลการจอง (Booking Data)
| ฟิลด์ (Field) | คำอธิบาย (Description) |
|---|---|
| **ชื่อผู้จอง** | ชื่อลูกค้าที่ทำการจอง |
| **เบอร์โทร** | เบอร์โทรศัพท์สำหรับติดต่อลูกค้า |
| **วัน-เวลาที่จอง** | วันที่และเวลาที่ลูกค้าจะเข้ามาใช้บริการ |
| **จำนวนคน** | จำนวนที่นั่งที่ลูกค้าต้องการ |
| **โซนที่นั่ง** | โซนที่ลูกค้าประสงค์จะนั่ง (อ้างอิงจาก Master Data) |
| **สถานะการจอง** | สถานะอัปเดต (เช่น รอดำเนินการ / ยืนยันแล้ว / ยกเลิก) |

### 5.2 ข้อมูลโต๊ะ/โซนที่นั่ง (Table/Zone Data)
| ฟิลด์ (Field) | คำอธิบาย (Description) |
|---|---|
| **รหัสโต๊ะ** | รหัสอ้างอิงของโต๊ะ (เช่น T01, T02) |
| **ชื่อโซน** | ชื่อโซนที่โต๊ะตั้งอยู่ (เช่น Indoor, Outdoor) |
| **จำนวนที่นั่งรองรับ** | ความจุสูงสุดของโต๊ะนั้นๆ (เช่น 2 คน, 4 คน) |
